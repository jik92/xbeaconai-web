# 爆款二创项目记录设计

## 目标

将爆款二创当前静态的“项目记录”抽屉改为真实、用户隔离、可恢复的项目工作区。用户可以搜索和筛选历史项目、分页浏览、重命名项目，并通过“继续创作”完整恢复上次的创作状态。

恢复范围包括：

- 当前工作阶段
- 商品、原始分镜视频和人像等项目输入
- 每条视频的解析结果和提示词版本
- 逐镜头生成任务、成功版本及当前选中版本
- 合成顺序、最终采用的原片或生成版本
- 最近一次合成任务及可播放结果

## 方案选择

使用 `workflowPhase=analysis` 的爆款二创根任务作为项目聚合根。根任务已经保存完整 `projectRequest`，提示词改写、逐镜头生成和合成任务均通过 `parentJobId` 与根任务关联，因此不新增重复的项目表。

新增爆款二创专用项目 API，由服务端聚合根任务和子任务。不能直接在前端复用通用 `/api/jobs`：该接口会把提示词工具、镜头生成和合成任务都当成独立记录，并需要前端执行 N+1 查询才能还原项目。

## 服务端数据模型

根分析任务继续保存不可变的初始请求与分析结果，并在 `values` 中增加可更新的工作区快照：

- `workspaceStage`：当前步骤，范围为 0～4
- `promptStates`：按原分镜 Asset ID 保存提示词、版本和当前版本
- `selectedShotAssets`：按原分镜 Asset ID 保存当前选中的原片或成功生成 Asset
- `composeOrder`：原分镜 Asset ID 的有序列表
- `composePreviewId`：合成预览当前选中的原分镜

快照只保存用户界面决策。任务状态、生成结果和合成结果仍以子任务记录为准，避免复制运行状态。

工作区更新只允许在根分析任务完成或部分完成后进行，防止分析 Worker 使用旧任务值覆盖快照。服务端合并允许字段后更新根任务，而不是信任前端覆盖整个 `values`。

## API

新增以下受 Bearer Token 保护的接口：

1. `GET /api/video-remix/projects`
   - 查询参数：`query`、`stage`、`page`、`pageSize`
   - 只返回当前用户的根分析任务
   - 返回项目名称、商品名称、当前阶段、状态、镜头进度、创建人、创建和更新时间
   - 服务端负责搜索、阶段筛选、分页和总数

2. `GET /api/video-remix/projects/{projectId}`
   - 返回根任务、全部项目子任务和归一化工作区快照
   - 返回可恢复的项目输入、解析结果、镜头生成历史和最近合成任务
   - 验证所有权，不暴露其他用户项目

3. `PATCH /api/video-remix/projects/{projectId}`
   - 支持独立更新项目标题或工作区快照
   - 标题去除首尾空白并限制长度
   - 校验阶段、提示词结构、源视频覆盖关系、选中版本归属和合成顺序

项目阶段由持久快照与真实任务状态共同决定。排队或处理中任务优先显示其运行阶段；成功合成显示“已完成”；其余情况使用已保存的工作阶段。

## 前端数据流

项目记录抽屉打开时查询专用项目列表。搜索和阶段筛选由“查询”触发并重置到第一页；翻页请求服务端真实分页数据。抽屉显示加载、空结果、错误和重试状态。

点击“继续创作”后：

1. 请求项目详情。
2. 从 `projectRequest` 恢复商品、原始视频、人像、需求和项目名称。
3. 从分析结果与 `promptStates` 恢复每条视频的提示词及版本。
4. 从镜头生成子任务恢复历史成功版本和进行中状态。
5. 恢复用户选择的镜头版本、合成顺序和最近合成结果。
6. 切换到持久化阶段并关闭抽屉，继续现有轮询。

提示词编辑、AI 改写应用、镜头版本选择、合成顺序调整和阶段切换后保存工作区快照。使用短延迟合并连续输入，离散操作立即保存。保存失败显示明确提示并保留本地编辑，允许下一次操作重试。

项目标题在记录抽屉内就地编辑；确认后调用 PATCH，成功时同步当前打开项目的标题。

## 素材与异常处理

- 项目详情恢复时使用稳定的 Asset ID 和鉴权内容地址，不保存 Blob URL。
- 已删除或无权访问的素材不能被静默替换；详情接口返回缺失 Asset ID，前端显示“素材已不存在”。
- 某个生成版本丢失时回退为对应原片，并提示用户重新选择；不能跨镜头匹配其他生成版本。
- 项目列表和详情均保持 owner 隔离。
- 工作区 JSON 损坏时服务端返回安全默认值，并保留真实任务数据用于恢复。
- 运行中的分析、镜头生成和合成任务继续使用现有轮询，不因打开项目记录重复提交任务。

## UI 约束

保留现有项目记录抽屉的单层中文标题、表格列和整体结构，不新增副标题或说明文案。搜索框、进度筛选、查询按钮、分页和“继续创作”改为真实交互。状态使用克制的语义标签，重命名使用现有铅笔入口。

## 测试与验收

- Store：只列出根项目、用户隔离、搜索、阶段筛选、分页、允许字段更新。
- API：项目列表和详情、非法项目 ID、跨用户访问、标题校验、工作区结构校验、跨镜头版本拒绝。
- 恢复：项目输入、提示词版本、镜头历史、选中版本、合成顺序和合成结果完整还原。
- 前端：真实查询参数、加载/空/错误状态、重命名、继续创作和保存触发。
- 回归：AI 解析、提示词工具、逐镜头生成、混合合成和媒体预览保持可用。
- 运行相关单元测试、全量单元测试、`bun run typecheck` 和 `bun run build`；不运行 E2E。
