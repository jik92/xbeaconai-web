# MediaKit 字幕擦除与画质增强设计

## 目标与范围

将现有字幕擦除和画质增强页面从通用 Mock 流程切换为火山引擎 AI MediaKit 真实异步任务。

首版能力：

- 字幕擦除：精细化自动字幕擦除。
- 画质增强：成本和时延优先的极速版。
- 输入：用户素材库中的视频，或经现有素材上传流程加入素材库的视频。
- 输出：永久保存到当前用户默认素材文件夹的 MP4，并在任务结果中预览和下载。

首版不提供字幕区域框选、目标分辨率、目标帧率或增强场景参数。现有界面中的这些控件没有可靠映射到选定接口，
继续展示会造成能力误导。

## 方案选择

采用共享 MediaKit Provider 加两个专用 Worker Handler。Provider 统一实现鉴权、提交、查询、状态归一化、错误脱敏和结果下载；
Handler 分别负责字幕擦除和极速画质增强的业务参数、进度文案、持久化与素材入库。

不扩展通用创建 Handler，因为该流程允许 Mock fallback，且不具备这类长任务所需的恢复和幂等语义。API Server 也不直接执行或等待
MediaKit，继续遵守 Server 只投递 BullMQ、Worker 执行真实异步任务的进程边界。

## 数据流

1. 前端通过现有附件选择器选取视频；本地文件先按现有直传流程成为用户私有素材。
2. API 校验模块开放状态并创建持久任务，BullMQ 消息只携带任务 ID。
3. Worker 根据 `assetId` 和 `ownerUserId` 读取素材，验证所有权和视频 MIME 类型。
4. Worker 为现有私有 TOS 对象生成短期签名读取 URL，不重复上传源视频。
5. MediaKit Provider 提交任务，将 `providerTaskId`、状态、提交时间和截止时间写入 SQLite。
6. Worker 轮询统一任务查询接口，并把排队、处理状态映射到任务阶段和进度。
7. 完成后下载短期结果 URL，将 MP4 写入当前用户默认素材文件夹对应 TOS 前缀并创建素材记录。
8. 任务结果引用持久素材接口，可用于鉴权预览和下载。

## 恢复、取消与幂等

- 任务已有 `providerTaskId` 时只恢复查询，不再次提交，防止重复计费。
- Worker 重启后由现有恢复机制重新投递未完成任务，并从 SQLite 持久状态继续。
- 用户取消后停止本地轮询；若 MediaKit 没有可用取消接口，则记录 `providerCancelState=unsupported`，并明确上游任务可能继续。
- 云端已完成但结果下载失败时保留任务 ID，重试只重新查询和下载结果。
- 只有结果成功持久化到 TOS 和素材表后，任务才进入 `succeeded`。

## 错误与安全

- 未配置 Key、权限不足、余额不足、限流、无效素材、上游失败、轮询超时和下载失败映射为结构化错误。
- 两个专用 Handler 禁止切换到 Mock，失败必须显示真实来源和原因。
- 日志、错误和任务结果不得包含 API Key 或带查询签名的完整 URL。
- Worker 不信任表单内的用户或 owner 标识，素材查询始终使用任务持久的 `ownerUserId`。
- MediaKit 返回 URL 有时限，不能直接作为永久 artifact URL。

## 前端

- 字幕擦除表单显示视频选择和“精细化自动擦除”只读能力说明，移除假框选。
- 画质增强表单显示视频选择和“极速增强”只读能力说明，移除未映射参数。
- 打开画质增强功能开关。
- 任务列表继续复用现有轮询/SSE、结果弹层、鉴权媒体预览和下载。
- 来源显示 `real`，Provider 显示 `volcengine-ai-mediakit`。

- Worker 单元测试覆盖素材所有权、幂等提交、重启恢复、取消、永久入库和禁止 Mock fallback。
- 前端覆盖视频必填、精简参数、功能开放和真实结果显示。
