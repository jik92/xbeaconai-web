# Header 全局搜索设计

## 范围

实现 AppShell Header 中间搜索框和快捷键按钮。搜索覆盖当前可见页面入口、用户任务、素材、商品、音色和人像。

## 交互

- 点击搜索框、点击快捷键按钮或按 `Command+K` / `Ctrl+K` 打开搜索面板并聚焦输入框。
- 输入关键词后按分组显示页面、任务和资产结果。
- 支持方向键移动高亮项、Enter 打开、Escape 关闭；点击面板外关闭。
- 选择页面或任务后进入对应工具页；选择资产后进入对应资产页并附带定位参数。
- 没有关键词时显示当前可见页面入口；结果为空时显示紧凑空状态。

## 数据与复用

提取共享 `GlobalSearch` 组件和纯函数搜索索引。AppShell 继续作为全局导航状态所有者，仅传入遵循菜单显示设置的页面入口。任务和资产通过现有查询 API 按面板打开状态加载；单类数据失败不会阻塞页面入口或其他类别结果。

搜索不新增后端接口，不复制 DTO，不显示 `menuFeatures=false` 且用户未主动设为显示的页面。

## 视觉

沿用 Header 现有紧凑搜索框，快捷键区域改为可点击按钮。结果面板使用白底、细分割线、`text-xs`/`text-2xs` 和现有图标，不增加标题副文案或卡片式装饰。

## 验证

增加索引、过滤和快捷键相关单元测试；运行字体检查、TypeScript 类型检查和生产构建。默认不运行 E2E。
