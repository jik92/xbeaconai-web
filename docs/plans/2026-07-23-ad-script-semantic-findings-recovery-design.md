# 口播脚本语义风险输出容错

## 问题

口播脚本模型偶尔把 `semanticFindings` 返回为字符串数组，而服务端只接受对象数组。Zod 校验失败后，原始 Schema 错误会进入任务失败信息，用户既无法理解，也无法完成已付费的生成流程。

## 设计

- 为生成、调优和重新评分共用一份完整的 `semanticFindings` JSON 契约，明确每项字段和空数组规则。
- 标准对象继续严格校验；非空字符串风险项转换为一条 `warning`，保留原始内容作为风险说明，并补齐稳定的规则 ID、空摘录和人工复核建议。
- 其他非法结构仍触发模型修复重试，避免静默接受无法解释的数据。
- 第二次输出仍不合法时，对用户返回可理解、可重试的“模型返回格式不稳定”提示，不暴露 Zod 内部错误；Worker 继续沿用现有失败退款和部分成功策略。
- 底层 Provider、模型、计费和数据库结构保持不变。

## 验证

- 单元测试覆盖标准对象、字符串风险项和非法结构。
- 运行相关单元测试、Biome、TypeScript 类型检查和生产构建。
