# 爆款二创逐镜头视频生成设计

## 目标

在爆款二创的“分镜校对”阶段复用 AI 创作提交组件，让每个原分镜能够独立提交真实视频生成任务、保留历史成功版本，并将用户选择的生成版本或原视频作为下一步合并输入。

## 方案选择

采用爆款二创专用分镜生成链路：前端复用共享 `PromptWorkbench`，服务端创建 `video-remix` 专用任务，Worker 复用 Seedance 生成边界并将结果保存为素材库 Asset。

不直接使用通用 `ai-generate` 任务。通用任务结果是 Artifact，无法直接满足素材所有权、镜头关联、刷新恢复和后续 Asset 合并要求。

## 前端交互

- 顶部视频卡片继续切换当前镜头，并显示该镜头的排队、生成中、成功或失败状态。
- 分镜校对底部的静态提示词条替换为共享 `PromptWorkbench`。
- 当前镜头的提示词自动带入输入框；模型、画幅、清晰度、时长和参考模式与 AI 创作一致。
- 原分镜视频固定作为视频参考；商品图默认作为图片参考。用户可以替换图片参考，但不能解除原分镜与生成任务的关联。
- 提交只生成当前镜头，不影响其他镜头。
- 每个镜头显示所有成功生成版本，默认选中最新成功版本；用户可切换旧版本作为合并输入。
- 生成结果可直接播放。同一镜头允许重新生成，失败后也可再次提交。
- 未生成或生成失败的镜头继续使用原视频；已有选中成功版本的镜头使用生成视频，允许混合合并。

## API 与持久状态

新增爆款二创专用分镜生成接口，请求包含：

- 原始解析任务 ID
- 原分镜 Asset ID
- 提示词
- Seedance 模型、画幅、清晰度、时长和参考模式
- 用户选择的额外参考 Asset ID

服务端验证：

- 解析任务属于当前用户且已经完成或部分完成。
- 原分镜属于该解析任务并且是视频。
- 参考素材属于当前用户，类型和数量满足 Seedance 限制。
- 模型及参数来自当前已验证能力目录。

生成任务使用 `moduleId: video-remix`、`workflowPhase: shot-generation`，通过 `parentJobId` 关联解析任务，并在 values 中持久化原分镜 ID。

新增按解析任务读取分镜生成历史的接口。刷新页面后，前端可以恢复每个镜头的任务、状态和成功版本。

## Worker 与生成结果

新增专用 `video-remix-shot-generation` Handler，并注册在通用 fallback 之前。

Handler 复用现有 `SeedanceVideoJob`：

1. 读取并校验任务中已经持久化的镜头和参考素材。
2. 调用 Seedance 或显式 Mock 能力生成视频。
3. 校验生成文件可解码。
4. 上传 TOS 并创建当前用户的视频 Asset。
5. 将 Asset ID、鉴权预览地址、生成参数和 lineage 写入任务结果。

失败只影响当前镜头；错误保持结构化并允许按错误类型重试。真实、Mock 或混合来源必须在任务和结果中明确显示。

## 合并数据流

合并请求使用有序映射，不再只提交原素材 ID：

```ts
type RemixComposeSource = {
  sourceAssetId: string;
  selectedAssetId: string;
};
```

- `sourceAssetId` 必须完整覆盖解析任务中的全部原分镜且不能重复。
- `selectedAssetId` 可以等于原分镜 ID。
- 如果选择生成结果，服务端必须验证该 Asset 来自当前解析任务下、对应原分镜的成功生成任务。
- Worker 按映射顺序读取 `selectedAssetId`，因此支持原视频与生成视频混合合并。

合并任务持久化最终映射，使项目记录可以还原实际使用的每个镜头版本。

## 测试与验收

- API：所有权、镜头归属、参数校验、非法跨镜头版本、历史查询。
- Worker：Handler 路由、生成结果保存为视频 Asset、失败隔离和 lineage。
- 前端：当前镜头提交、状态轮询、历史版本切换、刷新恢复和混合合并请求。
- 回归：原视频预览、提示词工具、多选、时间线排序和真实合并保持工作。
- 执行相关单元测试、完整单元测试、TypeScript 类型检查和生产构建；不运行 E2E。
