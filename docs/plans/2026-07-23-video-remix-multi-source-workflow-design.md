# 爆款二创多分镜视频工作流设计

## 目标

爆款二创的分镜视频支持一次选择或上传多条视频。每条视频独立进行真实 AI 解析和提示词校对，合并阶段允许调整顺序；只有用户点击“开始合并”后才创建真实合并任务和成片文件。

## 素材选择

- 最多选择 20 条视频，与现有 API Schema 上限一致。
- 再次选择时追加未重复素材，不覆盖已有列表。
- 列表保持选择顺序，每条视频可独立播放和删除。
- 至少保留一条视频才能提交 AI 解析。

## 独立解析

一个持久化批次 Job 保存全部视频素材标识，Worker 逐条执行媒体探测、语音转写和视频理解。每条视频产生独立 Markdown 提示词和状态，并通过结果数据绑定原素材 ID、文件名和错误信息。

成功条目可以立即用于后续校对；单条失败不清除其他结果，批次使用部分成功状态。取消请求在视频边界检查，后续不再调用模型。

## 校对界面

“提示词校对”和“分镜校对”顶部显示全部视频，点击切换当前视频。提示词文本、版本历史、智能检查、智能修改和换口播均按视频隔离。分镜校对的主预览和素材图片跟随当前视频。

## 合并成片

- 时间线展示所有原视频，默认顺序为选择顺序。
- 支持拖拽排序，并提供前移、后移按钮作为键盘可用替代。
- 点击“开始合并”前，主区域仅按当前顺序逐条预览原视频，不产生合并文件。
- 点击后提交独立的 `video-remix` 合并 Job。服务端重新验证素材归属和视频类型，Worker 使用现有 FFmpeg 标准化及拼接能力生成一个真实 MP4，保存到用户默认素材文件夹。
- 合并成功后主预览切换为成片，下载操作只对真实生成产物开放。

## 错误与验证

解析结果保留逐条错误；合并失败不影响原视频和校对结果，可再次提交。测试覆盖多选追加与去重、服务端归属校验、逐条解析、部分成功、视频切换、排序、未点击不合并和点击后真实合并任务。运行完整单测、类型检查和生产构建；按仓库规则不主动运行 E2E。
