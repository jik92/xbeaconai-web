# 紧凑认证页面设计

## 目标

统一登录、注册、忘记密码和设置密码四个流程，使用 `DESIGN.md` 的白色、近黑文字和紧凑产品风格。保留所有认证逻辑，仅重构视觉结构与组件实现。

## 布局

- 页面使用白色背景和居中窄卡片，不使用大面积渐变、厚重阴影或营销式双栏。
- 四种认证流程共用 `h-[480px] max-w-sm` 固定卡片外框；内容顶部对齐，流程切换时卡片尺寸、标题和字段起点保持稳定，避免页面抖动。
- 卡片使用 shadcn `Card` 的白色表面、细边框、适度圆角和 `shadow-sm`，不恢复旧版装饰性渐变与大阴影。
- 顶部只显示 Logo、项目名和当前流程标题；移除副标题和底部 SQLite 说明。
- 登录与注册使用紧凑 ghost tabs；忘记密码和设置密码使用单行返回操作与标题。
- 表单控件保持 `h-9`，字段间距使用 4px spacing system，整体宽度控制在约 360px。

## 组件

- 复用 shadcn `Card`、`Input`、`Button`、`Label`，布局与状态使用 Tailwind。
- 密码显示按钮使用 Input 内的 ghost icon Button。
- 验证码输入和发送按钮保持同一行；测试环境返回验证码时继续在发送成功状态旁显示。
- 错误与成功状态使用小型语义色块，不增加说明性副标题。
- 删除 `account.css` 中登录页专用选择器，保留账号抽屉等其他功能样式。

## 验证

- 增加认证页面结构测试，确认四个流程共用 shadcn 组件且旧登录 CSS 不再存在。
- 运行格式检查、字体规范、相关单测、类型检查和构建；不运行 E2E。
