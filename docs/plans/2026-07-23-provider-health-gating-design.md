# Provider 检测持久化与功能门控设计

## 目标

将密钥 Doctor 结果持久化到 SQLite，并把“最近一次检测通过”作为真实 Provider 功能的启用条件。未检测、缺少配置、鉴权失败或超时都视为不可用；前端显示禁用状态，服务端同时拒绝提交。

## 状态模型

- Provider 使用稳定 ID：`aihubmix`、`volc-speech`、`tos`、`mediakit`。
- 每个 Provider 保存状态、展示名称、结果消息、延迟和检测时间。
- Doctor 每次检测全部 Provider，并覆盖保存全部结果。
- 新增、修改、删除或导入某个 Provider 的密钥时，立即删除该 Provider 的历史检测结果；必须重新检测。
- 检测状态无自动过期时间，直到密钥变化或再次检测覆盖。

## 功能映射

- AIHubMix：爆款二创、一键成片、广告脚本以及其他 AIHubMix 生成模块。
- 火山语音：声音克隆/语音合成任务。
- AI MediaKit + 火山 TOS：字幕擦除、视频增强。
- 火山 TOS：视频切片、视频提取、视频剪辑、视频混剪、素材直传，以及需要 TOS 中转的生成任务。
- `menuFeatures=false` 的功能继续遵循静态开放配置，不因 Doctor 状态而显示或开放。
- 资产浏览、历史任务查看和不依赖 Provider 的本地能力不受影响。

## 服务端

- 增加公开的功能可用性接口，返回每个模块和素材上传操作的 `enabled`、缺失 Provider 与禁用原因。
- 通用任务、专用 AI 工作流、MediaKit、TOS 直传和分享视频导入在创建任务前强制检查。
- 拒绝时返回结构化 `PROVIDER_NOT_VERIFIED` 错误，不创建任务、不扣创作点、不调用 Worker。
- Worker 保留现有运行期校验，防止任务入队后 Provider 状态变化造成静默降级。

## 前端

- 侧边栏保留静态开放功能，但 Provider 未通过时显示为灰色禁用项，不按 `menuFeatures=false` 的默认隐藏逻辑处理。
- 直接访问禁用模块时显示紧凑的不可用状态页。
- 素材库仍可浏览；只有上传、创建商品等 TOS 操作按钮禁用。
- 管理后台首次进入即读取持久化 Doctor 结果，检测后刷新；密钥变化后对应状态立即回到未检测。

## 验证

- 测试 Doctor 结果持久化、重启读取、失败覆盖和密钥变更失效。
- 测试 Provider 与模块映射、服务端门控和静态 `menuFeatures=false` 隔离。
- 测试侧边栏禁用、直接路由保护和素材上传按钮。
- 重新生成 migration、OpenAPI 与 SDK，运行数据库检查、单元测试、类型检查和生产构建；不运行 E2E。
