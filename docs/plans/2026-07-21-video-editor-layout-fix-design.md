# 视频剪辑布局修复设计

## 问题

- Remotion Player 固定占满预览宽度，竖屏视频按宽度计算出超高画面并覆盖工具栏和时间轴。
- 工具按钮没有单行 flex 约束，图标和文字在窄空间内换行重叠。
- 时间轴自身同时承担固定工具栏和横向滚动，宽轨道会干扰控制区布局。

## 方案

- 预览区使用 size container，播放器外框根据画面比例在可用宽高内执行 contain 计算；Player 只填充该外框。
- 工具按钮统一为不换行的 inline-flex，消息区收缩，缩放控件保持在右侧。
- 工具栏固定在时间轴顶部，刻度尺和片段轨道放入独立横向滚动容器。
- 保持现有三栏结构、剪辑操作、Remotion 预览和导出协议不变。

## 验证

- 检查竖屏和横屏素材不会突破预览区。
- 检查 1440×900 与 1024×768 下按钮、缩放控件和时间轴可用。
- 运行类型检查、构建和相关时间线测试。
