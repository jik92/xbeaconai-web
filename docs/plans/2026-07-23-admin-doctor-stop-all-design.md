# 管理后台 Doctor 与停止全部任务设计

## 目标

简化管理后台密钥编辑，提供不产生模型费用的 Provider 可用性 Doctor，并允许管理员一次停止当前全部排队和执行任务。页面遵循 `DESIGN.md` 的紧凑白色工作区风格，优先复用 Tailwind 和 shadcn 组件。

## 密钥管理与 Doctor

- 密钥按 Provider 分组；同一 Provider 的字段集中编辑，每组只保留一个保存操作。
- 保留 `.env.key` 导入入口和单项删除能力，但减少重复状态、按钮和装饰容器。
- 顶部提供“检测全部”按钮，不在保存后自动检测。
- Doctor 使用无模型费用的只读或鉴权探测：AIHubMix 读取模型列表，TOS 检查 Bucket，火山语音查询保留的不存在音色，AI MediaKit 查询保留的不存在任务。
- 对于使用不存在资源的探测，鉴权通过后的资源不存在响应视为 Key 可用；401、403、权限或资源授权错误视为不可用。
- 每个 Provider 返回 `available`、`missing`、`invalid` 或 `timeout`，包含耗时和脱敏后的简短消息。响应不得包含明文密钥、请求头或完整上游响应。

## 停止全部任务

- 管理员确认后取消所有 `queued` 和 `processing` 任务，但不暂停 BullMQ 队列，后续任务仍可正常提交。
- `queued` 任务立即标记为 `cancelled` 并从 BullMQ 移除。
- `processing` 任务写入 `cancelRequested` 和“正在取消”，由现有 Worker/Provider 取消路径收敛。
- API 返回排队取消数、执行中请求数和处理失败数，并保持重复调用幂等。

## 界面

- 管理页面使用全高白色内容区、hairline 分隔和紧凑间距，不使用外围卡片或副标题。
- 复用 shadcn `Button`、`Input`、`DataTable`；状态使用 Tailwind 的小型语义文本和图标。
- 队列工具栏增加克制的危险操作按钮，确认后执行并刷新任务列表。

## 验证

- Doctor 单元测试覆盖成功、缺少配置、鉴权失败、业务资源不存在和超时。
- 批量停止测试覆盖排队任务、执行任务、终态任务、队列删除失败和重复调用。
- 前端测试覆盖检测全部、Provider 级保存和停止确认入口。
- 运行 API 契约生成、相关单测、类型检查、格式检查和构建；不运行 E2E。
