# 爆款二创提示词 AI 工具设计

## 目标

将提示词校对阶段的“智能检查”“智能修改”“换口播”从占位交互升级为可配置的全屏 Modal，并通过独立 Worker 调用真实 AI 改写当前完整提示词。

## 交互

- 三个入口均打开接近参考图结构的全屏工作台：左侧固定配置栏，右侧展示当前完整提示词。
- 智能检查支持检查范围、参考模式、检查类型、修复规则和自定义要求。
- 智能修改支持预设场景、自定义指令、修改范围和参考模式。预设包括“美妆人脸美白”“人脸美白”“商品替换”，预设与自定义指令叠加生效。
- 换口播包含“修正口播”和“换口播”两个页签。修正模式补全或纠正口播字段；换口播模式按自定义要求重写口播内容。
- 提交后在 Modal 内展示任务阶段和进度。成功后用模型返回的完整提示词更新预览和主页面，并生成一个 AI 修改版本；失败时保留原提示词并显示错误。

## 服务端与 Worker

- API Server 只校验输入、验证源任务归属并投递 BullMQ Job，不直接执行模型调用。
- 新 Job 使用 `video-remix` 模块和独立的 `prompt-rewrite` workflow phase，由专用 Handler 处理。
- Worker 通过现有 Aihubmix Provider 调用真实 `deepseek-v4-pro`，不使用 Mock fallback。
- 模型输出使用结构化 JSON，至少包含完整 `prompt`、`summary` 和 `findings`；服务端校验后再写入任务结果。
- 改写必须保持 Markdown 主结构、分镜顺序、时长、商品事实及未被选中修改的字段。

## API 契约

- 新增受保护的爆款二创提示词工具任务接口。
- 请求包含源分析任务 ID、当前提示词、工具类型及工具配置；owner ID 只从登录态获取。
- 响应沿用现有 Job Schema。前端通过现有任务查询接口轮询。
- 契约修改后重新生成 OpenAPI 和 TypeScript SDK。

## 视觉与错误处理

- 参考截图的信息架构，但使用项目现有白色表面、hairline、圆角和 ink 主按钮，不引入新的高饱和品牌色。
- Provider 未配置、模型不可用、输出格式无效或任务失败时显示结构化错误，绝不静默切换到本地规则或 Mock。
- 关闭 Modal 不取消已提交任务；再次打开时可继续看到当前任务状态。

## 验证

- 单元测试覆盖三种工具配置、模型 Prompt 边界、输出解析、Handler 注册、API 契约和前端交互源码边界。
- 运行 OpenAPI/SDK 生成、目标单测、类型检查和生产构建。
- 使用已配置 Provider 实测模型列表和一次最小真实改写；若凭据或外部服务不可用，明确记录未验证原因。
