# 紧凑型文件上传组件设计

## 目标

将项目内分散的大型拖拽上传区收敛为统一、紧凑的文件上传入口，并保持 shadcn 基础组件的视觉语义。上传业务逻辑、文件校验、TOS 直传和素材库选择流程保持不变。

## 设计

- 新增共享 `FileUpload` 组件，组合现有 `Button`、原生文件输入和辅助文本。
- 默认使用紧凑横向布局：描边按钮负责选择文件，右侧显示文件名或状态，下方按需显示格式、大小及目标目录说明。
- 上传中禁用选择按钮并显示旋转图标；错误仍由各业务流程在原位置展示。
- 组件统一支持 `accept`、`multiple`、`disabled`、文件名、提示语和选择回调，不接管具体上传请求。
- `AttachmentPicker` 的本地上传页首先复用该组件，并移除原有固定高度、蓝色虚线大面板样式。
- 素材库上传弹窗复用同一组件，保留商品图片预览、上传进度和资产字段。
- 工具表单仍通过 `AttachmentPicker` 选择素材，不改变其素材库与本地上传双入口语义。
- 管理员密钥导入和视频编辑器顶部“添加”属于操作栏按钮，不使用带辅助信息的表单上传布局，暂不强制替换。

## 可访问性与状态

- 真实 `input[type=file]` 保留在 DOM 中，并由明确按钮触发。
- 选择按钮具备可配置的可访问名称；禁用状态同步作用于按钮和文件输入。
- 文件选择后立即向业务层传递 `File[]`，允许调用方自行清空、上传或展示预览。
