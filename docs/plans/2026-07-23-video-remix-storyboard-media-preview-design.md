# 爆款二创分镜校对媒体预览设计

## 目标

分镜校对阶段的“结果预览”必须展示用户当前选择的真实原始分镜视频、商品图片和人像图片，不再使用静态占位场景或空图片块。

## 视频预览

从当前 `source` 值提取素材 ID，通过 `/api/assets/{id}/content` 和现有 `AuthenticatedMedia` 加载私有视频。该阶段固定预览原始分镜视频，不把仅包含 Markdown 的解析任务结果误判为生成视频。没有视频或加载失败时显示明确状态。

## 图片预览

- 商品图片遍历 `selectedProduct.images`，通过每张图片已有的鉴权 URL 使用 `AuthenticatedMedia` 渲染。
- 已选人像继续使用人像库公开 URL，不向外部图片域名附加登录 Token。
- 人像加载失败时显示局部错误状态；未选择人像时不显示伪造图片。
- 分镜提示行与结果素材列表复用同一个人像加载组件。

## 结果操作

当前解析任务没有生成视频产物时，下载结果按钮保持禁用，避免把原始片源误认为生成结果。

## 验证

单元测试覆盖原始视频鉴权 URL、真实商品图片映射、人像错误处理、静态占位移除和无生成结果时的下载状态。运行相关测试、完整单测、TypeScript 类型检查和生产构建；按仓库规则不主动运行 E2E。
