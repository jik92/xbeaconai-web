# 口播脚本入口始终显示配置表单

## 背景

口播脚本页面会把最近一次项目 ID 写入 `localStorage`。再次从菜单进入页面时，只要该记录存在，页面就直接显示“AI 智能调优中”，用户无法从配置表单开始新的创作。

## 设计

- 每次进入 `/aigc/ad-script` 时都初始化为配置表单和第一步。
- 不再读取、写入或保留口播脚本项目 ID 的跨会话记录。
- 保留配置表单草稿持久化，避免用户输入意外丢失。
- 用户在当前页面提交生成后，仍按原流程进入调优页，并在完成后进入结果页。
- 不新增“继续上次任务”入口或任务恢复兼容逻辑。

## 验证

- 增加前端回归测试，确认页面源码不再包含活动项目的本地存储键，并仍以表单作为初始界面。
- 运行相关单元测试、TypeScript 类型检查和生产构建。
