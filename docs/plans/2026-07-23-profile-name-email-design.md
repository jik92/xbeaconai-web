# 个人资料名字与邮箱设计

## 目标

个人资料只允许修改名字和邮箱。移除前端的头像及头像文字概念；原头像展示位置改为显示用户名，用户名为空时显示手机号。

## 数据与接口

- 为用户表新增可空、唯一的邮箱字段，兼容现有没有邮箱的账号。
- 邮箱保存前去除首尾空格并统一为小写；允许留空，不进行验证码验证。
- 个人资料接口只接收 `displayName` 和 `email`，校验名字长度、邮箱格式和邮箱唯一性。
- 数据库保留旧 `avatarText` 字段，避免扩大迁移范围，但用户可见界面和个人资料接口不再使用它。
- 邮箱冲突返回明确的结构化错误，由前端显示“邮箱已被使用”。

## 界面

- 个人资料表单只显示“名字”和“邮箱”两个输入框。
- 顶部账号入口和账号摘要移除圆形头像展示，直接显示用户名；无用户名时回退到手机号。
- 保持现有紧凑布局和 Tailwind 字体系统，不增加副标题或说明文字。

## 验证

- 增加账号 Store/API 单元测试，覆盖邮箱保存、规范化、清空和唯一性冲突。
- 增加前端静态或组件测试，确认资料表单不再出现头像字段。
- 运行相关单测、类型检查和构建；除非用户另行要求，不运行 E2E。
