# 个人资料名字设计

## 目标

个人资料只允许修改名字。移除前端的头像、头像文字及邮箱概念；原头像展示位置改为显示用户名，用户名为空时显示手机号。

## 数据与接口

- 不新增邮箱字段或数据库迁移。
- 个人资料接口只接收 `displayName`，校验名字长度。
- 数据库保留旧 `avatarText` 字段，避免扩大迁移范围，但用户可见界面和个人资料接口不再使用它。

## 界面

- 个人资料表单只显示“名字”输入框。
- 顶部账号入口和账号摘要移除圆形头像展示，直接显示用户名；无用户名时回退到手机号。
- 保持现有紧凑布局和 Tailwind 字体系统，不增加副标题或说明文字。

## 验证

- 增加账号 Store/API 单元测试，覆盖名字保存和规范化。
- 增加前端静态或组件测试，确认资料表单不再出现头像字段。
- 运行相关单测、类型检查和构建；除非用户另行要求，不运行 E2E。
