# 一键成片参数面板补全设计

## 目标

补全 `/aigc/video-create` 新建项目左栏中的广告诉求、脚本风格和高级设置。所有标签均支持多选，选中项和自由输入必须保存到项目，并参与真实脚本与分镜生成，而不是只作为界面装饰。

## 交互设计

- 三组参数使用独立折叠面板：广告诉求、脚本风格、高级设置。
- 面板标题显示当前已选标签数量；没有选择时显示“可选”。
- 标签使用可点击的多选胶囊，提供明确的选中态、键盘焦点和触控反馈。
- 广告诉求包含营销目标、目标受众、用户痛点和产品利益点。
- 脚本风格包含主播角色、主播性别、内容风格、开场方式和结尾引导。
- 高级设置包含脚本题材、素材话题、营销手法、模板、敏感词和自定义要求，并承载视频模型、声音等生成参数。
- 折叠或切换项目不会丢失选择；历史项目恢复时还原全部字段。

## 数据与生成链路

- 在一键成片输入契约中增加结构化字符串数组和文本字段，不把多选项压缩进现有展示字符串。
- API、SQLite Store 和项目更新接口共同校验并持久化这些字段。
- AI 智能填写可返回并回填结构化推荐值；未知值会被规范化，避免污染预设选择。
- 脚本 Prompt 显式包含所有广告诉求、脚本风格、高级策略、敏感词和自定义要求。
- 分镜 Prompt 继承已确认的脚本风格与营销约束，保证脚本和镜头表达一致。

## 错误处理

- 多选字段设置数量与单项长度上限，非法输入由 API 返回结构化校验错误。
- 项目保存失败时保留当前本地编辑并显示错误，不静默丢弃选择。
- 旧项目缺少新字段时使用空数组和空字符串默认值，保持向后兼容。

## 验证

- 单元测试覆盖输入解析、落库恢复和生成 Prompt。
- API/SDK 重新生成并通过 TypeScript 类型检查。
- Playwright 覆盖桌面端 `1440x900` 与平板端 `1024x768` 的展开、多选、计数和历史恢复。
- E2E 注册账号使用每个测试唯一身份，避免并发冲突。
