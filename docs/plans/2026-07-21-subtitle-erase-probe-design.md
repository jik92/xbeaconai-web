# 字幕擦除 API 验证脚本设计

## 目标

提供一个 Bun 命令行脚本，使用本地视频验证火山引擎的真实字幕擦除能力。首个测试素材为
`/Users/aijianji/Desktop/视频裂变.mp4`。

## 方案

脚本接收本地视频路径，检查媒体文件后将其暂存到现有私有 TOS Bucket。脚本为暂存对象创建短期签名读取地址，
再复用 `.env` 中现有的火山访问凭证调用视频点播或智能处理的精细化字幕擦除接口。脚本轮询异步任务，下载最终视频到
`artifacts/api-tests/subtitle-erase/`，并生成不包含密钥和完整签名 URL 的 JSON 报告。

优先验证 AK/SK 签名的专用视频字幕擦除接口。若现有凭证缺少产品权限，脚本应明确报告所缺权限或凭证，不得切换到
通用图片模型、逐帧处理或 Mock。MediaKit Bearer API 作为后续可选接入方式，只有配置专用 API Key 后才启用。

## 数据流

1. 校验输入路径并使用现有 FFmpeg 封装读取视频元数据。
2. 将源视频上传到 TOS 隔离暂存前缀，并生成有时限的私有读取 URL。
3. 提交字幕自动识别与精细化擦除任务。
4. 以有限间隔轮询任务，直到成功、失败或超时。
5. 下载结果视频，检查响应状态、文件大小和媒体可读性。
6. 写入脱敏报告，并在结束时删除 TOS 暂存源文件。

## 错误与安全

- 不打印 AK、SK、Bearer Key 或包含签名参数的完整 URL。
- 外部接口不可用、权限不足、额度不足和参数错误分别输出可定位的错误。
- 使用超时与有限轮询，避免脚本永久挂起。
- 无论任务成功或失败，均尝试清理本次创建的 TOS 暂存对象。
- 不删除输入视频；远端结果 URL 过期前下载到本地。

## 验证标准

- 脚本可通过 `bun scripts/test-subtitle-erase.ts <video-path>` 执行。
- 实际提交服务时明确记录 provider、task ID、状态和输出文件。
- 成功时使用 FFprobe 验证输出视频可读；失败时报告真实服务错误且不产生 Mock 结果。
