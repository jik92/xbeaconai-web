# 爆款二创提示词校对视频预览设计

## 问题

提示词校对页的顶部视频切换卡片使用静态图标，没有显示各自视频的真实画面。多选两个或更多分镜后，用户无法通过卡片画面辨认对应视频。

## 交互设计

- 顶部每个视频切换卡片直接显示该视频的真实首帧，不再使用静态视频图标。
- 卡片缩略视频静音且不显示控制条，点击卡片仍只切换当前视频、解析状态和对应提示词。
- 提示词内容区保持原有“版本列表、提示词文档”两栏布局，不额外插入播放器。
- AI 解析失败只影响提示词结果，不影响对应卡片加载原视频首帧。

## 数据与错误处理

- 不新增服务端接口或持久化字段，每张卡片使用自身素材 ID 和现有 `AuthenticatedMedia` 加载授权视频。
- 视频首帧加载失败时在对应卡片内显示显式错误状态，不回退到错误或无鉴权的 URL。
- 切换视频不重置各视频已经生成的提示词版本。

## 验证

- 更新单元测试，断言每个视频切换卡片挂载自身素材的鉴权缩略视频。
- 覆盖卡片点击继续使用 `activeSourceId` 切换对应提示词。
- 运行相关单元测试、TypeScript 类型检查和生产构建；不运行 E2E。
