# 密钥管理表格设计

## 目标

将管理后台密钥管理从 Provider 分组表单改为统一表格，降低布局噪音并让配置、检测和操作状态可以横向比较。

## 布局

- 复用共享 `DataTable`，每个密钥占一行。
- 固定列为 Provider、密钥、当前状态、新值、检测结果和操作。
- 顶部紧凑工具栏继续承载 `.env.key` 导入和“检测全部”，不增加卡片或说明区。
- 表格填满剩余内容高度并在内部纵向滚动，不产生横向滚动条。

## 交互

- 新值直接在行内输入；每行独立保存，不再按 Provider 分组保存。
- 删除操作保留二次确认，并与保存操作位于行尾。
- Doctor 结果按 Provider 返回；同一 Provider 的各密钥行展示相同检测结果。
- 保存、删除或导入后清空旧 Doctor 结果，避免展示过期状态。

## 验证

- 更新管理页结构测试，确认密钥管理复用 `DataTable` 且不再渲染 Provider 分组区块。
- 运行格式检查、相关单测、类型检查和生产构建；不运行 E2E。
