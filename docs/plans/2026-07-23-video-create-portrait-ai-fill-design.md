# 一键成片人像选择与 AI 参数填充设计

## 目标

修复一键成片工作台的两处断点：人像图片必须从现有预置人像库选择；“AI 填充参数”使用 `gpt-5.4-mini` 分析商品图片和可选人像图片，并用分析结果覆盖全部脚本表单参数。

## 方案

采用专用人像选择器，不复用普通素材附件选择器。前端从现有 `/portraits.json` 读取 1,125 份预置人像，支持搜索、筛选、预览和单选。项目只保存稳定的 `portraitId`，服务端根据受控人像目录解析图片地址，禁止前端提交任意外部图片 URL。

AI 分析继续复用一键成片的 `analyze` 异步任务和 BullMQ Worker。Worker 收集全部商品图片；已选择人像时，同时加入对应人像图片。模型调用通过现有 AIHubMix 凭据和 OpenAI 兼容接口完成，模型固定为 `gpt-5.4-mini`，要求返回严格 JSON。

## 数据流

1. 用户点击人像图片的“添加”，打开专用人像库弹窗并选择一份人像。
2. 前端把 `portraitId` 连同其他一键成片输入保存到项目。
3. 用户点击“AI 填充参数”，API 保存当前输入并投递 `analyze` Job。
4. Worker 从用户私有商品素材解析本地图片，并从受控人像目录解析可选人像 URL。
5. `gpt-5.4-mini` 分析图片并返回结构化推荐。
6. 服务端使用 Zod 校验、别名归一化和枚举白名单清理结果。
7. 通过校验后，推荐结果覆盖所有脚本参数；商品图片、人像、音色、视频模型等资源选择保持不变。
8. 前端轮询项目并一次性显示完整新参数。

## 覆盖语义

每次 AI 填充都覆盖全部可分析表单参数，包括产品名称、核心卖点、广告场景、时长、段落数、创作要求、脚本风格、营销诉求、目标人群、人群痛点、产品利益点、讲述人设定、内容风格、开场方式、结尾引导、脚本主题、素材主题、营销方式、模板、敏感词和自定义要求。

AI 不修改商品图片、人像选择、音色、视频模型、画幅、字幕开关和画面/口播优先级。模型无法判断的字段使用 Schema 默认值或空值，避免残留上一次填写内容。

## 错误处理与来源记录

- 缺少商品图片时前端保持现有阻止提交逻辑。
- 人像编号不在受控目录时，API 返回结构化参数错误，不执行任务。
- 图片不存在、下载失败、总大小超限、模型未配置、模型返回无效 JSON 或超时时，任务明确失败并保留原表单输入。
- 只有完整结果通过校验后才原子覆盖项目输入，不写入部分结果。
- 分析阶段记录 `real`、`aihubmix-gpt-image-analysis` 和 `gpt-5.4-mini`，不静默切换模型或 Mock。

## 验证

- 单元测试覆盖人像编号解析、非法编号拒绝、模型请求的多图消息结构、结构化结果归一化和全量覆盖语义。
- 一键成片 Worker 测试覆盖商品图加可选人像图的分析路径，以及失败时不修改原输入。
- 运行相关单测、`bun run typecheck` 和 `bun run build`。
- 按仓库约定不运行 E2E，页面交互由用户手动验证。
