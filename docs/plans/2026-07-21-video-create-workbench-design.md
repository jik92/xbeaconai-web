# 一键成片两栏工作台设计

## 背景与目标

`/aigc/video-create` 当前复用通用任务列表和新建弹窗，只能提交“主题/文案、素材来源、比例、音色、字幕”等扁平参数，通用 Worker 还可能把未映射能力静默降级为 Mock。此次改造将它升级为独立、可恢复的一键成片项目工作台，参考 `/Users/jik1992/Desktop/公司/一键成片_产品设计稿`：左侧填写项目参数，右侧通过“脚本 / 分镜”两个 Tab 审核产出物。

目标流程为：

1. 上传商品图片并由 AI 推荐参数。
2. 使用真实文本或多模态模型生成分段脚本。
3. 用户逐段审核、编辑或换一版。
4. 使用真实文本模型生成结构化分镜。
5. 每个分镜独立生成视频，失败镜头可重试或上传替代视频。
6. 全部分镜就绪后合并最终视频。

生产链路禁止静默 Mock。自动化测试中的耗时视频生成与合并可以显式使用 Mock；商品分析、参数推荐、脚本和分镜等文本类能力必须通过真实接口测试。

## 页面架构

进入 `/aigc/video-create` 后直接展示工作台，不再先显示任务列表或弹出通用新建窗口。

### 左栏：新建项目

桌面宽度约 360–400px，独立滚动，底部固定当前阶段主按钮。参数包括：

- 商品图片上传、删除、解析状态和 AI 填充参数。
- 可选人像图片。
- 广告场景。
- 产品名称和最多八条核心卖点。
- 视频时长、分镜段数和配音语速。
- 可折叠的广告诉求、脚本风格和高级设置。
- 高级设置中的 Seedance 模型、系统音色或个人音色、画面比例和字幕策略。

AI 填充根据商品图片推荐产品名称、核心卖点、广告场景、视频时长、分镜数、广告诉求和脚本风格，但只回填表单，不自动产生费用或推进生成。

Seedance 默认使用 Fast；高级设置可切换 Standard、Mini 和 Fast，只展示通过当前环境真实能力检测的模型。配音默认使用推荐系统音色，也可选择音色库中的个人音色。

### 右栏：产出物

右栏占据剩余宽度，顶部固定“脚本 / 分镜”Tab、数量徽标和“生成记录”按钮，内容区独立滚动。

脚本 Tab 提供：

- 总字数和预计时长。
- 复制全文和重新生成。
- 按分镜段落展示的脚本卡片。
- 每段独立编辑、保存、换一版，并实时重算字数和时间。
- 脚本审核完成后的“下一步：生成分镜”。

分镜 Tab 提供：

- 分镜段落、素材、配音和字幕列。
- 每镜头的排队、生成中、成功、失败和替代素材状态。
- 单镜头重新生成、上传替代视频和预览。
- 口播优先/画面优先策略。
- 所有镜头就绪后开放“合并视频”。

“生成记录”使用右侧抽屉，不离开当前页面。记录覆盖草稿、生成中、待审核、部分失败和已完成项目；选择后恢复全部参数、脚本、分镜和任务状态。

### 响应式

- `1440×900` 为主要验收尺寸，两栏各自滚动，主操作始终可见。
- `1024×768` 仍保持两栏，压缩左栏和次要操作文字。
- 更窄尺寸改为顶部“参数 / 产出物”切换，避免不可用的窄双栏。

## 持久化模型

新增专用的一键成片项目聚合，不把可编辑状态塞入通用 Job 的字符串值中。

### 项目

保存 owner、项目名称、商品与人像素材引用、完整参数、模型、音色、当前阶段、总体状态、乐观锁版本和时间戳。

状态流转为：

`draft → script_generating → script_review → storyboard_generating → storyboard_review → composing → completed`

失败状态保留最后一个稳定阶段和结构化错误，不删除已成功结果。

### 脚本段落

保存项目、顺序、段落类型、正文、预计时长、当前版本和更新时间。换一版创建新版本；失败时保留旧版本。保存时带预期版本，避免多页面并发覆盖。

### 分镜

每个分镜对应一个脚本段落，保存文案、生成提示词、参考素材、配音与字幕开关、Seedance 上游任务、视频素材、状态、尝试次数和结构化错误。分镜可独立重试，或使用用户上传的视频替代。

## API 与任务流

使用 `@hono/zod-openapi` 定义专用契约并重新生成 SDK。建议接口边界如下：

- 创建、查询、更新和列出一键成片项目。
- 上传完成后的素材关联。
- 商品分析与 AI 参数推荐。
- 生成脚本、保存段落、单段换一版。
- 生成分镜方案。
- 生成或重试单个分镜视频、上传替代视频。
- 提交最终合并。
- 查询阶段任务和取消尚未提交上游的任务。

每个产生外部副作用的接口使用幂等键；编辑接口使用预期版本。API Server 仅校验、持久化和投递 BullMQ，不执行后台生成。

新增专用 Worker Handler，并在通用 fallback 之前注册：

- 商品多模态分析与参数推荐。
- 分段脚本结构化生成。
- 单段脚本改写。
- 结构化分镜生成。
- 单镜头 Seedance 提交、轮询、核对和结果保存。
- 配音、字幕和最终 FFmpeg 合并。

所有阶段进度和结果回写 SQLite。Server 与 Worker 重启后根据数据库和上游任务 ID 恢复；不得以进程内 Map 作为业务真相。

## 错误与恢复

- 商品分析或脚本生成失败时保留参数，并允许从当前阶段重试。
- 单段换一版失败时不覆盖当前可用文案。
- 单镜头失败不取消其他镜头；失败行提供重试和上传替代视频。
- 存在失败或生成中的镜头时禁用合并，并显示未就绪数量。
- 合并失败只重试合并，不重新生成镜头。
- 余额不足、模型不可用、素材失效、版本冲突和接口超时使用独立错误码。
- 后台任务执行期间阻止同阶段重复推进，但允许读取和恢复项目。
- 生产环境禁止 Mock 回退；错误必须明确暴露真实失败原因和是否可重试。

## 测试与验收

### 自动化测试

- 单元测试覆盖参数 Schema、状态机、版本冲突、幂等、失败镜头重试、合并门槛和 owner 隔离。
- API/Worker 集成测试使用临时 SQLite 和测试队列，覆盖创建、恢复、脚本、分镜、部分失败和合并状态回写。
- Playwright 在 `1440×900` 和 `1024×768` 覆盖两栏布局、脚本编辑、Tab 切换、镜头重试、生成记录恢复和合并按钮状态。
- 耗时视频生成与合并在普通自动化中使用显式 Mock Provider，任务及产物必须标记 `mock`，且只允许测试环境启用。

### 真实能力测试

- 商品分析、AI 填参、脚本生成、单段换一版和分镜结构化生成必须实际调用真实模型，并断言结构化输出。
- 提供独立真实视频冒烟脚本，以最小单镜头验证 Seedance 提交、轮询、结果下载和最终合并；不纳入普通 CI。
- 不以 SDK 安装、环境变量存在或静态注册表代替真实测试结果。

### 交付基线

契约变化后运行：

```bash
bun run api:spec
bun run api:generate
bun run typecheck
```

最终至少运行相关单元测试、`bun run build` 和两种视口的 E2E。真实文本能力测试必须通过；未运行的真实视频冒烟测试需在交付说明中明确列出。

## 非目标

- 不改造 `/aigc/video-remix` 爆款二创流程。
- 不在通用 `ModulePage` 中继续增加 `video-create` 特殊判断。
- 不把测试 Mock 作为生产降级策略。
- 不在本次设计中引入多人协作、时间线精剪或复杂音轨编辑器。
