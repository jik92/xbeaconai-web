# 口播脚本设计系统布局重构

## 目标

将 `/aigc/ad-script` 的配置、生成进度和结果界面统一到 `DESIGN.md`，优先使用 Tailwind 与已有 shadcn 组件，并移除页面中用户可见的 DeepSeek 模型描述。底层模型、API、计费和任务状态流程保持不变。

## 页面结构

- 页面使用全高白色产品工作区，内容宽度限制在约 1200px。
- 标题区只保留“口播脚本”和必要操作，不显示英文 eyebrow、描述性副标题或模型标签。
- 配置流程保留三步结构，使用紧凑步骤导航、hairline 分隔和一致的内容卡片。
- 进度页保留真实任务状态、进度、变体列表、返回和取消操作。
- 结果页保留摘要、变体切换、脚本编辑、评分、合规、版本、导出、重新评分和继续调优。

## 组件与视觉

- 按钮、卡片、输入框、标签和下拉框优先使用现有 shadcn `Button`、`Card`、`Input`、`Label`、`NativeSelect`。
- 主要操作使用近黑色 pill；辅助操作使用 outline 或 ghost；卡片使用白色表面、hairline 边框和 `rounded-xl`。
- 产品页面采用 `text-sm` 默认字号，页面标题不超过 `text-2xl`，不使用业务 CSS 中的原始字号或硬编码品牌色。
- 场景与角色使用响应式选择卡，风格和长度使用紧凑 pill，桌面多列、平板两列、小屏单列。
- 专用 CSS 仅保留 Tailwind 不适合表达的必要动画，不再维护平行的按钮、表单和卡片视觉系统。

## 文案与行为

- 移除 `deepseek/deepseek-v4-pro`、“固定 DeepSeek 模型”和“固定模型”等用户可见描述。
- “AI 智能调优中”等任务状态文案保留。
- 页面每次进入仍从配置表单开始；生成后的当前会话继续进入进度页和结果页。
- 不修改 Provider、模型请求参数、创作点计算、API DTO 或 Worker 行为。

## 验证

- 更新静态 UI 回归测试，验证页面使用共享 shadcn 组件、初始界面固定为表单且不含可见 DeepSeek 文案。
- 运行相关单元测试、Biome、typography 检查、TypeScript 类型检查和生产构建。
- 按仓库规则不运行 E2E；由用户在本地检查实际页面。
