# BYOK 主密钥自动初始化设计

## 背景

管理后台读取凭证和 `bun run byok:import-env` 都依赖 `BYOK_ENCRYPTION_KEY`。旧 `.env` 没有该字段时，导入会失败，管理接口返回 `BYOK_UNAVAILABLE`。

## 设计

`scripts/import-byok-env.ts` 在导入前检查 `.env` 与当前进程配置：

- 已配置至少 32 字符的主密钥时直接使用，不修改 `.env`。
- 完全缺少主密钥时生成 32 字节随机值，以 64 位十六进制字符串原子追加到 `.env`，文件权限保持为仅当前用户可读写。
- 存在但少于 32 字符时拒绝自动覆盖，并提示管理员手动修正，避免已加密数据因密钥变化而无法解密。
- 日志只说明密钥已生成或已存在，永远不输出密钥明文。

生成后，脚本使用新主密钥打开凭证库并继续导入白名单内的 Provider Key。导入完成后提示重启 Server 和 Worker，使已经启动的进程加载主密钥。

## 验证

- 在不包含主密钥的临时 `.env` 中运行脚本，确认自动写入、导入成功且日志不包含明文。
- 再次运行确认不会覆盖主密钥。
- 短主密钥必须失败且文件保持不变。
- 运行类型检查和相关单元测试，并用实际项目 `.env` 执行一次导入。
