# 抖音 API 测试子进程隔离

## 目标

仅修复抖音 API 集成测试污染开发数据库的问题，不改全局测试启动方式或 Server 架构。

## 方案

- 原抖音 API 集成用例移动到不被默认测试扫描的独立目录。
- 单元测试目录保留一个包装用例；包装用例创建临时数据目录，并用独立 Bun 子进程运行原集成用例。
- 子进程启动时通过环境变量注入 `YAOZUO_DATA_DIR`，确保 `server/env.ts` 首次加载前就指向临时数据库。
- 子进程结束后清理临时目录，并把失败输出带回父测试。
- 开发库只清理可明确关联到测试账号的等待恢复任务；操作前创建数据库备份。

## 验证

- 运行包装用例并确认通过。
- 对比运行前后开发数据库文件和分享导入任务数量，必须保持不变。
- 运行类型检查和相关格式检查。
