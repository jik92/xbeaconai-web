# 一键成片分镜配音与字幕设计

## 目标

让分镜编辑中的配音、字幕和素材操作形成真实可用链路：配音与字幕默认开启，配音可以试听，字幕展示时间轴，素材可以从素材库替换或本地上传，并在最终合成时应用开关。

## 方案

- 每个分镜保存独立的音频产物 ID 和字幕时间轴 JSON；现有配音、字幕布尔字段继续作为合成开关，数据库默认值保持为 `true`。
- 分镜生成任务使用该段当前脚本文案调用 AIHubMix `tts-1`，首版使用默认 `alloy` 音色。现有“音色素材”是参考音频，不作为不存在的 TTS speaker ID 使用。
- TTS 完成后探测真实音频时长，按中文标点切句，并按字符权重分配时间，生成可稳定复现的字幕时间轴。
- 配音开启时，最终合成用分镜 TTS 替换视频音轨；字幕开启时，将该段时间轴烧录到分镜视频，再拼接所有分镜。
- 素材替换不清除已生成的配音和字幕；重新生成分镜会重新生成与当前脚本一致的配音和时间轴。

## 界面

- 表头保留配音和字幕总开关，用于批量更新所有分镜；行内保留独立开关。
- 配音列展示该段口播文本、原生音频播放控件和重新生成后的状态；没有音频时显示明确的待生成状态。
- 字幕列逐条展示 `开始秒数 ～ 结束秒数` 和对应字幕内容；关闭后保留内容但使用弱化样式。
- 素材列提供两个明确入口：“素材库”打开库选择，“本地上传”直接进入上传选择；两者最终都调用已有替换接口。
- 继续采用 `DESIGN.md` 的紧凑产品界面、白色表面、hairline 分隔、ink 主按钮和 shadcn 控件，不引入页面专属 CSS。

## API 与数据

- `video_create_shots` 新增 `audio_asset_id` 和 `subtitle_cues_json`。
- 分镜响应增加 `audioArtifactId` 与 `subtitleCues`。
- 增加项目级批量更新分镜开关接口，服务端校验项目归属并在事务内更新。
- 音频通过现有鉴权 artifact 地址播放，不暴露本地文件路径。

## 错误与一致性

- 配音开启时 TTS 失败会让该分镜任务明确失败，不静默退化为视频原声或 Mock。
- Worker 更新分镜视频、音频、字幕和状态时保持同一持久化路径；重试会覆盖当前分镜的产物引用，不追加重复时间轴。
- 关闭配音或字幕不删除产物，便于重新开启且避免重复生成。

## 验证

- Store/API 单测覆盖默认开启、音频和字幕字段、批量开关以及用户资源隔离。
- Worker 单测覆盖字幕切句与时间分配。
- 运行数据库迁移检查、相关单测、类型检查、构建和完整 CI；不运行 E2E，由用户手动验证页面。
