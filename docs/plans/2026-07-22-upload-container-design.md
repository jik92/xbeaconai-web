# 可拖拽上传容器设计

## 目标

将当前紧凑文件输入升级为统一的虚拟上传容器。组件支持点击选择、拖拽文件、自动或确认后上传、真实进度以及按文件类型预览，同时保持 shadcn 的边框、圆角、状态色和表单语义。

## 组件边界

- 上传容器负责文件选择、拖拽状态、上传状态、进度条、错误区和预览布局。
- 业务页面负责文件校验之外的元信息、目标文件夹和具体上传 API。
- 组件以受控属性接收进度、上传结果和错误，避免与素材库或单一 API 耦合。
- 统一上传结果包含文件名、MIME、大小和可鉴权访问地址；预览复用现有 `AuthenticatedMedia`。

## 交互状态

1. 等待文件：显示上传图标、操作标题和格式提示。
2. 拖拽悬停：使用主色边框和浅色背景，提示松开即可添加。
3. 上传中：禁用重复选择，显示文件名、真实百分比和进度条。
4. 上传完成：图片显示缩略图，视频和音频显示播放控件，其他文件显示文件摘要。
5. 上传失败：保留文件与错误信息，提供重试和重新选择。

容器维持紧凑高度，不恢复此前 250px 的大面积装饰上传区。拖拽区域应清晰可发现，但不使用夸张图标或脱离表单体系的颜色。

## 上传策略

- 创作工具和附件选择器使用自动上传：文件选择或拖入后立即调用上传 API，成功后回填业务字段。
- 素材库、音色和商品上传需要元信息，先生成本地预览；用户确认后在同一容器显示上传进度。
- 单文件直接使用底层上传进度。
- 多文件并发上传时按各文件大小加权汇总总进度。
- TOS 直传和服务端 FormData 上传都使用可观测上传进度，不展示模拟百分比。

## 文件预览

- 图片：完整比例缩略图。
- 视频：带播放控件的视频预览。
- 音频：原生音频控制条。
- 其他文件：文件图标、名称、类型和大小。
- 本地待上传文件使用短生命周期对象 URL；上传成功后切换为服务端返回的鉴权资源地址并及时释放本地 URL。

## 错误与可访问性

- 拒绝不符合 `accept` 的文件并给出明确提示。
- 拖拽容器支持键盘触发，真实文件输入保留在 DOM 中。
- 进度使用 `role="progressbar"` 和完整 ARIA 数值。
- 上传状态使用 `aria-live`；错误后允许重试或移除文件。
- 组件卸载时释放所有本地对象 URL。

## 验证

- 单元测试覆盖文件约束、拖拽、进度、状态和不同媒体预览。
- 端到端测试覆盖素材上传与附件自动上传。
- 运行 `make ci`、类型检查、生产构建以及桌面和 Tablet 相关用例。
