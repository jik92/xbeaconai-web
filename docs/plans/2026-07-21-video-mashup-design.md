# 视频混剪设计

## 目标

将现有关闭状态、仅能走通用 Mock Worker 的“视频混剪”升级为真实可用的自动批量混剪功能。

首版支持建立多个视频组，每个成片从每组选择一个视频，按视频组顺序拼接，批量输出差异化 MP4 并保存到用户指定的 TOS 素材文件夹。

首版不包含脚本语义匹配、黄金模板、字幕、贴纸、背景音乐和品牌包装。

## 信息架构

- 保留模块 ID：`video-mashup`
- 保留路由：`/tools/video-mashup`
- 在 `web/app/config.ts` 中正式开放功能。
- 保留现有任务列表页结构，新建任务使用专用双栏混剪配置界面。
- 新增专用 `job-video-mashup.ts`，不再由通用 Mock Worker 处理。

现有视频分割、视频剪辑和其他模块不受影响。

## 页面交互

### 视频组

新建任务左侧用于管理视频组：

- 默认创建两个空视频组。
- 支持添加、删除视频组，最多 10 组。
- 每组具有可编辑组名。
- 每组通过素材选择器从当前用户素材库中多选视频。
- 素材选择器支持按文件夹筛选，只展示视频。
- 每组最多选择 20 个视频。
- 已选素材显示缩略图、名称和时长，可单独移除。

提交时至少需要两个非空组，每组至少包含一个视频。

### 输出设置

右侧包含：

- 任务名称。
- 组合模式：最多结果数、最大差异化。
- 分辨率：720P、1080P。
- 最多生成数量：1–20。
- 保存位置：用户真实素材文件夹。

页面实时显示视频组数、可用素材数、理论组合总数和预计生成数量。

移除首版没有真实行为的黄金模板、画面类型、分镜贴纸、自动采纳和全局贴纸。

## 组合算法

### 最多结果数

按照视频组顺序计算笛卡尔积，并使用稳定的词典序输出组合，达到用户设置的生成数量后停止。

### 最大差异化

先构造候选笛卡尔组合，再以确定性贪心算法选择结果：

1. 第一条采用稳定词典序中的第一条。
2. 后续每次选择与已选组合的最小汉明距离最大的候选。
3. 距离相同时，优先选择全局素材使用次数更少的候选。
4. 仍相同时按稳定词典序决定。

该算法保证同一输入产生相同结果，并尽量降低相邻和批次内的素材重复。

理论组合数使用各组素材数乘积计算，并做安全上限处理，避免大规模笛卡尔积占用过多内存。实际输出数为理论组合数、用户数量和 20 三者的最小值。

## 音视频规则

- 视频组顺序决定片段拼接顺序。
- 每个成片保留第一段素材的原声。
- 后续片段静音。
- 不添加背景音乐。
- 所有源素材先标准化为统一分辨率、30 FPS、H.264 视频流。
- 第一段输出 AAC 音频；没有原声时生成静音音轨。
- 后续片段生成与视频等长的静音 AAC 音轨，确保最终拼接流结构一致。
- 画面按目标比例等比缩放并使用黑色留边，不裁切主体。
- 输出为 H.264/AAC MP4，启用 faststart。

目标尺寸：

- 720P：1280×720
- 1080P：1920×1080

## Server 校验

前端提交版本化 JSON 配置，视频组只包含组 ID、组名和素材 ID，不传长期 URL。

Server 必须校验：

- Schema 版本。
- 组数为 2–10。
- 每组素材数为 1–20。
- 同组内没有重复素材。
- 组合模式和分辨率合法。
- 输出数量为 1–20，且不超过理论组合数。
- 所有素材存在、属于当前用户并且 MIME 类型为视频。
- 目标文件夹存在且属于当前用户。

Server 固化标准化配置到 Job values，再投递 BullMQ。

## Worker 数据流

1. 再次读取并校验 Job 配置、素材所有权和目标文件夹。
2. 创建独立临时目录。
3. 对所有去重后的素材执行一次下载与 `ffprobe`。
4. 每个源素材只标准化一次，供所有组合复用。
5. 根据组合模式生成确定性组合计划。
6. 逐个拼接成片，上传 TOS 并创建素材记录。
7. 每完成一个成片就回写 artifact、组合键和进度。
8. 全部完成后标记成功；部分组合失败时标记部分成功；全部失败时标记失败。
9. 最终清理临时目录。

任务阶段包括素材校验、素材下载、素材标准化、组合规划、批量渲染和 TOS 保存。上传阶段使用 TOS SDK 的真实进度回调，并在阶段文案中显示当前成片序号。

## 幂等、取消与恢复

- 每个组合使用基于有序素材 ID 的稳定组合键。
- Job 结果数据记录已成功组合键。
- 重试时跳过已成功且素材记录仍存在的组合，避免重复输出。
- 每个下载、标准化、渲染和上传阶段检查取消状态。
- 取消时停止后续组合，已成功保存的成片予以保留。
- 取消或失败时清理临时文件和未完成的 TOS 分片上传。
- Worker 崩溃恢复后根据持久化结果继续剩余组合。

## 结果展示

混剪批次结果按卡片展示全部视频 artifact：

- 缩略预览或视频预览。
- 文件名和组合序号。
- 单个下载。
- 进入所在素材文件夹。
- 批次摘要显示成功数、失败数和目标文件夹。

部分成功时明确展示失败组合数量，不隐藏错误。

## 错误处理

- 无效配置在 Server 投递前返回结构化 422 错误。
- 素材或文件夹越权统一返回不可用，不泄露资源是否属于其他用户。
- FFprobe、FFmpeg、TOS 或源素材下载失败记录具体阶段和可重试性。
- 单个组合失败不阻断其余组合。
- 能力缺失时明确标记本地 FFmpeg 或 TOS 不可用，不切换 Mock。

## 测试与验收

### 单元测试

- 笛卡尔积、稳定顺序和数量上限。
- 最大差异化距离、素材使用均衡和确定性。
- 空组、重复素材、非法数量和理论组合数。
- API 素材归属、视频 MIME 和文件夹归属校验。
- Worker 标准化复用、第一段原声、后续静音、部分成功、取消和恢复幂等。

### 端到端测试

- 功能入口已开放。
- 添加、删除视频组。
- 按文件夹多选视频并移除素材。
- 组合摘要随输入实时变化。
- 提交有效任务并展示批次结果。
- 检查 1440×900 和 1024×768。

- `make ci`
