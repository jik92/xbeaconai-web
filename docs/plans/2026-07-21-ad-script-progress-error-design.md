# 口播脚本进度页错误误报修复设计

## 问题

口播脚本进度页在项目查询没有错误时仍调用错误格式化函数。该函数对 `undefined` 返回默认文案，导致正常生成和调优期间固定显示“操作失败，请稍后重试”。SQLite 中的任务和变体仍处于真实处理状态，因此这是前端展示误报。

## 方案

- 仅当 React Query 提供真实 `error` 时调用错误格式化函数。
- 保留页面操作错误以及 API、Worker、模型返回的真实错误文案。
- 在口播脚本 E2E 中进入真实进度页，断言正常排队或处理中不显示错误反馈。

## 验证

- 运行口播脚本 Playwright 桌面端与 Tablet 用例。
- 运行 TypeScript 类型检查和生产构建。
