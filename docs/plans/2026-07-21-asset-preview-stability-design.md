# 素材预览稳定性修复设计

## 问题

素材表格加载媒体元数据后会更新本地状态。列定义依赖该状态和每次渲染都会变化的回调，导致 React Table 的单元格渲染组件身份变化。预览节点因此反复卸载，鉴权 Blob URL 被回收并重新请求，表现为持续闪烁。

## 修复

- 将素材预览单元格抽成文件级稳定组件，组件类型不随列定义变化。
- 使用 `useCallback` 固定元数据记录、上传和删除回调。
- 列定义继续读取最新尺寸和时长，但状态变化只更新必要属性，不更换预览组件类型。
- 保留 `AuthenticatedMedia` 现有 Blob URL 生命周期，不增加全局媒体缓存。
- 保留元数据回写、视频延迟播放和预览尺寸适配行为。

## 验证

- 增加回归测试，模拟多个预览依次加载元数据，确认已加载媒体节点不会重新挂载。
- 检查媒体内容接口请求不会因其他素材的元数据更新而重复触发。
- 运行相关单元测试、TypeScript、生产构建及素材库桌面/平板 E2E。
