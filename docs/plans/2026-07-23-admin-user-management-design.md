# 管理后台用户管理设计

## 目标

在管理后台增加用户管理，支持查询用户、管理员自定义创作点充值、停用账号和恢复账号。停用账号是可恢复操作，不删除用户关联数据。

## 页面

- 管理后台增加“用户管理”页签，复用共享 `DataTable`。
- 表格列为用户名、手机号、创作点、状态、注册时间、更新时间和操作。
- 顶部使用紧凑的用户名/手机号搜索和状态筛选，不增加卡片、副标题或说明区。
- 充值使用紧凑 Modal，只保留中文标题、正整数创作点输入和确认操作。
- 活跃用户显示“充值”和“注销”；已停用用户显示“恢复”。管理员不能注销自己的账号。

## 服务端

- 增加管理员用户分页查询接口，支持关键字和状态筛选。
- 增加管理员充值接口，使用幂等键避免重复到账。
- 新增管理员充值流水表，记录目标用户、管理员、充值点数、到账后余额和时间，不复用用户端套餐订单。
- 增加用户状态更新接口。停用时在同一事务中将状态改为 `disabled` 并撤销该用户全部有效会话；恢复仅允许从 `disabled` 回到 `active`。
- 所有接口经过现有管理员鉴权，不允许管理员停用自己。

## 数据与错误

- 用户、任务、素材、普通充值订单和管理员充值流水在停用后全部保留。
- 自定义充值只接受安全范围内的正整数，并校验到账后余额不超过 JavaScript 安全整数范围。
- 用户不存在、状态冲突、幂等冲突和管理员自停用返回结构化错误。

## 验证

- 增加 AccountStore 单元测试，覆盖分页、幂等充值、停用、会话撤销、恢复和管理员保护。
- 更新管理页结构测试，确认用户表格和紧凑充值 Modal。
- 重新生成数据库 migration、OpenAPI 和前端 SDK，并运行数据库检查、相关单测、类型检查和生产构建。
- 不运行 E2E。
