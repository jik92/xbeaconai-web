# 实用工具：视频提取与视频剪辑设计

## 目标

在侧边栏新增独立的“实用工具”分类，并提供两个全新页面：

- 视频提取：从公开直链或 `yt-dlp` 可解析的公开分享页下载视频，并保存到用户选择的 TOS 素材文件夹。
- 视频剪辑：提供基于 Remotion 的单轨剪辑工作台，支持添加、预览、切分、相邻合并、删除、撤销/重做、缩放和导出。

现有“AI 工具箱 → AI视频分割”及其 `video-cut` 任务链路保持不变。两个新功能使用独立模块 ID、页面、API、持久化记录和 Worker Job。

## 信息架构

侧边栏新增一级分类“实用工具”，包含两个并列入口：

- `/utilities/video-extract`：视频提取
- `/utilities/video-editor`：视频剪辑

菜单开关继续集中在 `web/app/config.ts`。侧边栏菜单偏好升级存储版本，迁移时为已有用户补充新分类和新入口，不覆盖其原有排序与隐藏偏好。

## 总体架构

### 视频提取

API Server 负责请求校验、用户文件夹鉴权、任务持久化和 BullMQ 投递。Worker 负责 URL 解析、下载、媒体探测、TOS 上传、素材记录创建、进度回写和临时文件清理。

下载支持两种来源：

1. 可直接访问的视频文件 HTTP/HTTPS URL。
2. `yt-dlp` 可解析且无需登录、验证码、付费或绕过 DRM 的公开分享页。

### 视频剪辑

前端使用 Remotion Player 预览。前后端共享一份版本化时间线 Schema。Worker 使用该时间线快照和 Remotion Renderer 导出 MP4，再将结果上传 TOS 并登记到素材库。

剪辑草稿持久化到 SQLite。导出时固化时间线快照并投递独立 BullMQ Job，因此关闭页面不会中断导出，继续编辑也不会改变正在渲染的结果。

Server 不执行下载或渲染，Worker 不承载 HTTP API。

## 视频提取交互

页面采用现有任务型页面结构：顶部显示标题、说明和“新建任务”，下方显示任务列表。

新建任务仅包含两个必填输入：

- 视频 URL
- 目标存储文件夹

任务状态包括排队、解析、下载、校验、上传、完成、失败和取消。完成后可预览、下载或进入对应素材文件夹。文件名优先采用来源标题，并经过安全清洗和冲突处理。

登录限制、验证码、私有内容、付费内容、DRM、非视频内容、体积超限和网络错误必须显示明确、可操作的错误，不静默降级或绕过限制。

## 视频剪辑交互

编辑器按参考设计稿构建为独立工作台：

- 顶栏：页面标题、添加视频、导出。
- 左栏：已添加的源视频列表。允许多个源视频，但首版时间线为单轨顺序结构。
- 中央：黑底 Remotion Player，支持播放、暂停和拖动定位。
- 右栏：无选择时显示提示；选择片段后显示名称、源入点、源出点和时长，并允许精确修改入点、出点。
- 工具栏：撤销、重做、切分、合并、删除和缩放。
- 时间线：显示缩略图片段、播放头、选中态和多选，内容区可横向滚动。

编辑规则：

- 切分：在播放头位置把当前片段拆成两个。
- 合并：仅允许合并时间线上相邻的选中片段；合并是编辑分组操作，不修改源文件。
- 删除：从时间线移除片段，可撤销。
- 首版不支持自由拖拽排序、多轨、滤镜、字幕、变速和转场。
- 草稿自动保存，重新进入页面可恢复。

导出弹窗包含目标文件夹、文件名和分辨率。输出保持源画面比例，使用 30 FPS、H.264 MP4。Worker 使用 Remotion Renderer 渲染并上传结果。

在 `1440×900` 下完整展示三栏。在 `1024×768` 下压缩左右栏，右侧属性栏在空间不足时收为可展开抽屉，时间线保持横向滚动和可用触控目标。

## 时间线数据模型

时间线采用版本化共享契约，至少包含：

- Schema 版本
- 项目 ID、所有者和项目名称
- 源素材 ID 列表及必要媒体元数据
- 片段 ID、源素材 ID、源入点、源出点和时间线顺序
- 当前导出宽高、帧率和总时长
- 修订号与更新时间

自动保存使用修订号避免旧请求覆盖新草稿。Server 必须重新验证素材所有权、片段边界、片段顺序和总时长，不能信任浏览器计算结果。

## 数据流

### 提取任务

1. 浏览器提交 URL 和目标文件夹 ID。
2. Server 校验协议、文件夹所有权和功能开放状态，创建 Job 并投递 BullMQ。
3. Worker 在独立临时目录中解析并下载。
4. `ffprobe` 验证文件包含视频流并读取媒体元数据。
5. Worker 上传到用户文件夹的 TOS 前缀并创建素材记录。
6. Worker 回写结果、来源标记、进度和通知，最后清理临时目录。

### 剪辑与导出

1. 用户从素材库添加视频或复用素材直传流程上传视频。
2. 编辑器使用素材短期鉴权地址交给 Remotion Player 预览，并自动保存时间线。
3. 用户提交导出，Server 校验项目并固化时间线快照。
4. Worker 下载归属用户的源素材到临时目录。
5. Remotion Renderer 使用本地素材和共享 composition 渲染 MP4。
6. Worker 上传结果、创建素材记录、回写任务状态并清理临时文件。

## 安全与错误处理

- 仅允许 HTTP/HTTPS URL，拒绝携带不支持协议的请求。
- DNS 解析和每次重定向后都阻止 localhost、环回、链路本地、内网和保留地址，防止 SSRF。
- 配置下载超时、最大重定向数、最大文件体积和任务取消检查。
- `yt-dlp` 不接收用户 Cookie，不执行绕过登录、付费或 DRM 的参数。
- 不信任扩展名或 Content-Type，下载后必须通过 `ffprobe` 验证视频流。
- 目标文件夹、源素材、剪辑项目和导出结果都必须执行 owner 隔离。
- 失败或取消时清理本地临时文件及未完成的 TOS 分片上传。
- Remotion 渲染失败保留结构化错误和可重试性，不静默切换为 FFmpeg 或 Mock。

## 测试与验收

### 单元与集成测试

- URL 协议、SSRF、重定向、超时、体积限制、非视频内容和取消。
- 目标文件夹与素材的所有权隔离。
- 时间线 Schema、切分、相邻合并、删除、撤销/重做和边界校验。
- 草稿修订冲突、自动保存和导出快照一致性。
- Worker 进度、失败、取消、恢复、TOS 上传和素材登记。
- Remotion 代表性帧和短片段渲染检查。

### 端到端测试

- “实用工具”分类及两个新入口可见。
- 创建视频提取任务并观察状态。
- 添加素材、播放定位、切分、合并、删除、撤销和导出。
- 检查 `1440×900` 和 `1024×768` 两种视口。

### 交付基线

运行相关测试、`make ci`、`bun run typecheck`、`bun run build` 和 `bun run e2e`。Remotion 渲染、TOS、Redis、`yt-dlp` 或 FFmpeg 若受本机环境限制，交付时明确列出未验证项与原因。
