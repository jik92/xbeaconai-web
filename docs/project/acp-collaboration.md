# ACP 协作与任务记录

> 最后更新：2026-07-21

ACP（AI Collaboration Protocol）可用于需要计划、执行、验证、审查或审计留痕的协作任务。

当前仓库没有可追溯的 `acp_template/` 协议源，也没有项目规则将 ACP 绑定为强制流程。因此，本页仅保留通用边界：只有宿主或项目规则明确提供协议路径、revision 和 fingerprint 后，才按指定协议创建和审核任务记录。

## 内容边界

```text
<protocol-template>/  已绑定时的只读协议、模板与校验工具
<workspace>/task_records/  某次任务的计划、证据、审查和归档
docs/                长期有效的项目认知、功能需求和 Bug 解决方案
```

任务记录保存执行事件、原始证据和审查历史；`docs/` 保存可复用的架构事实、产品决策和问题根因。不要持续双写同一份进度或逐条命令输出。

## 已绑定协议时的最小实践

1. 在协议目录外创建唯一任务记录，明确目标、范围、约束和验收项。
2. 为验收项建立计划、验证方法和可追溯证据。
3. 审查范围、回归和每项验收的闭环；将审查结论归档。
4. 仅将长期有效结论提升至 `docs/`，并保留任务记录的稳定引用。

不得在任务记录或文档中保存密码、Token、个人数据或生产敏感信息。
