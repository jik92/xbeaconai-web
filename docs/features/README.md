## 建档流程

1. 复制 `_template.md`，命名为 `FEAT-0001-short-name.md`，编号不可重复。
2. 完成当前行为、范围、用户流程、验收标准和影响分析。
3. 对照 [功能地图](../project/feature-map.md)，说明对 `web/`、`server/`、`worker/`、`shared/`、SQLite migration 和生成 SDK 的影响。
4. 将文档登记到索引；实现、测试、发布过程中持续维护同一份文档。

开发准入条件：目标/非目标明确，影响已分析，验收可验证，未决问题不会改变实施方向。

若宿主或项目规则已绑定 ACP，受治理任务在文档中只记录任务路径、ID 或不可变归档引用；实时事件、原始证据和执行日志保存于任务记录，不与知识库重复。
